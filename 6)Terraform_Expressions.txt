# Terraform Expressions:
------------------------
    - Terraform expressions are used to assign values to variables, resource arguments, and output values.
    - They can include literals, references to other resources, and built-in functions.
    - Expressions are evaluated at runtime and can be used to create dynamic configurations.
    - Examples of Terraform expressions include:
    - String interpolation: `"Hello, ${var.name}!"`
    - Arithmetic operations: `var.count + 1`
    - Conditional expressions: `var.environment == "production" ? "prod" : "dev"`
    - Function calls: `length(var.list)`
    - Terraform also supports complex expressions that can combine multiple operations and functions to achieve more advanced configurations.   
    - It is important to understand how Terraform evaluates expressions and the order of operations to avoid unexpected results in your configurations. 
    - When using expressions, it is crucial to ensure that the referenced variables and resources are defined and available at the time of evaluation to prevent errors during the Terraform plan and apply stages. 
    - Terraform expressions can also be used in modules to create reusable and flexible configurations, allowing for greater modularity and maintainability in your infrastructure code.    
    - Overall, mastering Terraform expressions is essential for creating efficient and effective infrastructure as code configurations, enabling you to leverage the full power of Terraform's capabilities in managing your infrastructure.    

    # Terraform Operators:
    ----------------------
        - Terraform supports various operators that can be used in expressions to perform operations on values.
        - These operators include:
        - Arithmetic operators: `+`, `-`, `*`, `/`, `%`
        - Comparison operators: `==`, `!=`, `>`, `<`, `>=`, `<=`
        - Logical operators: `&&`, `||`, `!`
        - Assignment operators: `=`, `+=`, `-=`, `*=`, `/=`, `%=`
        - Terraform also supports the use of parentheses `()` to control the order of operations in expressions, allowing for more complex calculations and logic to be implemented in your configurations.
        - It is important to understand the precedence of operators in Terraform to ensure that expressions are evaluated correctly and produce the expected results. 
        - When using operators in Terraform expressions, it is crucial to ensure that the operands are of compatible types to avoid type errors during evaluation. 
        - Terraform operators can be used in various contexts, including variable assignments, resource arguments, and output values, providing flexibility and power in your infrastructure as code configurations. 
        - By mastering Terraform operators, you can create more dynamic and efficient configurations, allowing you to manage your infrastructure with greater precision and control.    

    # Using Expressions with Lists and Maps in Terraform:
    ------------------------------------------------------
        - Terraform allows you to use expressions with lists and maps to create dynamic configurations.
        - For lists, you can use expressions to manipulate and access elements, such as:
        - `var.list[0]` to access the first element of a list.
        - `length(var.list)` to get the number of elements in a list.
        - `concat(var.list1, var.list2)` to combine two lists into one.
        - For maps, you can use expressions to access values and manipulate the map, such as:
        - `var.map["key"]` to access the value associated with a specific key in a map.
        - `keys(var.map)` to get a list of all keys in a map.
        - `values(var.map)` to get a list of all values in a map.
        - You can also use expressions to perform operations on lists and maps, such as filtering, mapping, and reducing, using built-in functions like `filter`, `map`, and `reduce`.  
        - When using expressions with lists and maps, it is important to ensure that the data structures are properly defined and that the expressions are correctly formatted to avoid errors during evaluation. 
        - By leveraging expressions with lists and maps, you can create more flexible and dynamic configurations in Terraform, allowing you to manage your infrastructure more effectively and efficiently. 

        # examples of each expressions type with lists and maps:
        --------------------------------------------------------
            - String interpolation with lists: `"The first item is ${var.list[0]}"`
            - Arithmetic operation with lists: `length(var.list) + 1`
            - Conditional expression with maps: `var.map["environment"] == "production" ? "prod" : "dev"`
            - Function call with lists: `concat(var.list1, var.list2)`
            - Accessing map values: `var.map["key"]`
            - Getting keys from a map: `keys(var.map)`
            - Getting values from a map: `values(var.map)`
            - Filtering a list: `filter(var.list, item -> item != "exclude")`
            - Mapping a list: `map(var.list, item -> upper(item))`
            - Reducing a list: `reduce(var.list, 0, (acc, item) -> acc + length(item))` 
            - These examples demonstrate how to use expressions with lists and maps in Terraform to create dynamic and flexible configurations, allowing you to manage your infrastructure more effectively.    

    # Terrafor for Expressions:
    ----------------------
        - Terraform provides a rich set of built-in functions that can be used in expressions to perform various operations on values, including lists and maps.
        - Some common built-in functions for expressions include:
        - `length()`: Returns the number of elements in a list or the number of keys in a map.
        - `concat()`: Combines multiple lists into a single list.
        - `keys()`: Returns a list of all keys in a map.
        - `values()`: Returns a list of all values in a map.
        - `lookup()`: Retrieves a value from a map based on a specified key, with an optional default value if the key is not found.
        - `join()`: Concatenates a list of strings into a single string, with an optional separator.
        - `split()`: Splits a string into a list of substrings based on a specified separator.
        - `upper()`: Converts a string to uppercase.
        - `lower()`: Converts a string to lowercase.
        - `replace()`: Replaces occurrences of a substring in a string with another substring.
        - These built-in functions can be used in expressions to manipulate and transform data in your Terraform configurations, allowing you to create more dynamic and flexible infrastructure as code. 
        - When using built-in functions in Terraform expressions, it is important to understand the expected input and output types for each function to ensure that your expressions are correctly formatted and produce the desired results. 
        - By leveraging the built-in functions in Terraform, you can create more powerful and efficient configurations, enabling you to manage your infrastructure with greater precision and control.

        # Example codes of FOR expressions with list and map:
        ------------------------------------------------------
            - Using `for` expressions with lists: 
            ```
            variable "list" {
              type = list(string)
              default = ["item1", "item2", "item3"]
            }

            output "upper_list" {
              value = [for item in var.list : upper(item)]
            }
            ```

            - Using `for` expressions with maps:
            ```
            variable "map" {
              type = map(string)
              default = {
                key1 = "value1"
                key2 = "value2"
                key3 = "value3"
              }
            }

            output "upper_map_values" {
              value = { for key, value in var.map : key => upper(value) }
            }
            ```
            - These examples demonstrate how to use `for` expressions with lists and maps in Terraform to create dynamic configurations that manipulate and transform data, allowing you to manage your infrastructure more effectively.    
    
    # Terraform Splat Expressions:
    --------------------------------
        - Terraform splat expressions are a powerful way to access and manipulate multiple elements in a list or map at once.
        - Splat expressions use the `*` operator to iterate over a list or map and apply an expression to each element.
        - For lists, you can use splat expressions to access properties of each element, such as:
        - `var.list[*].property` to access the `property` of each element in the list.
        - For maps, you can use splat expressions to access values based on keys, such as:
        - `var.map[*].value` to access the `value` of each key in the map.
        - Splat expressions can also be used to perform operations on multiple elements, such as filtering, mapping, and reducing, using built-in functions like `filter`, `map`, and `reduce`.
        - When using splat expressions, it is important to ensure that the data structures are properly defined and that the expressions are correctly formatted to avoid errors during evaluation. 
        - By leveraging splat expressions in Terraform, you can create more efficient and dynamic configurations, allowing you to manage your infrastructure with greater precision and control. 
        - Example of splat expressions with lists:
        ```
        variable "list" {
          type = list(object({
            name = string
            value = string
          }))
          default = [
            { name = "item1", value = "value1" },
            { name = "item2", value = "value2" },
            { name = "item3", value = "value3" }
          ]
        }
        output "names" {
          value = var.list[*].name
        }
        ```
        - Example of splat expressions with maps:
        ```
        variable "map" {
          type = map(object({
            value = string
          }))
          default = {
            key1 = { value = "value1" }
            key2 = { value = "value2" }
            key3 = { value = "value3" }
          }
        }
        output "values" {
          value = var.map[*].value
        }
        ```
        - These examples demonstrate how to use splat expressions with lists and maps in Terraform to access and manipulate multiple elements at once, allowing you to create more dynamic and efficient configurations for managing your infrastructure.
